2  Análisis velocidad

2.1 Objetivo y datos

En este documento vamos a:

  • Leer un CSV con resultados de una carrera de 1500m.
  • Extraer los splits cada 100m (columnas 100m, 200m, …, 1500m).
  • Construir representaciones comparables entre corredores.
  • Agrupar corredores por estrategia (forma del pacing) con clustering jerárquico.
  • Resumir e interpretar cada grupo y etiquetar automáticamente estrategias.

La idea clave es separar dos conceptos:

  • Nivel de rendimiento: quién es más rápido en términos absolutos.
  • Estrategia (pacing): cómo distribuye el esfuerzo a lo largo de la carrera (salida fuerte, ritmo constante, kick final, etc.).

Para agrupar por estrategia, compararemos la forma del perfil, no el tiempo absoluto.

2.4.1 Manejo simple de valores faltantes (NA)

dist() no maneja bien NA. Si existieran, usamos una imputación muy simple: reemplazamos cada NA por la media de su columna (tramo).

Esto no es “la verdad”, pero evita que el cálculo falle y suele ser aceptable si hay pocos NA.

if (anyNA(X)) {
  col_means <- colMeans(X, na.rm = TRUE)
  for (j in seq_len(ncol(X))) {
    X[is.na(X[, j]), j] <- col_means[j]
  }
}

2.5.2 Distancia por “forma de pacing” (z-score por corredor)

Para estudiar estrategia, queremos ignorar que uno sea globalmente más rápido que otro y fijarnos en el patrón relativo (salida / ritmo / kick).

Para eso, estandarizamos cada fila (cada corredor):

Restar la media de sus splits

Dividir por su desviación estándar

row_z <- function(v) {
  s <- sd(v, na.rm = TRUE)
  if (is.na(s) || s == 0) return(rep(0, length(v)))
  (v - mean(v, na.rm = TRUE)) / s
}

2.6 Clustering por estrategia (pacing)

Aquí usamos Xshape (perfil normalizado por corredor), que es la representación más alineada con “estrategia”.

2.6.0.1 Clustering jerárquico (Ward)

Calculamos distancias euclídeas entre perfiles normalizados.

Aplicamos hclust con método ward.D2.

d <- dist(Xshape, method = "euclidean")
hc <- hclust(d, method = "ward.D2")

2.6.1 Elección del número de clusters con silhouette

Probamos k = 2..8 y elegimos el que maximiza la silhouette media.

k_grid <- 2:8

2.9 Etiquetado automático de estrategias

A continuación definimos métricas simples (pero interpretables) para “nombrar” estrategias.

2.9.1 Métricas de estrategia

Usamos Xshape (z-score por corredor). Definimos:

  • slope: promedio (últimos 400m) − promedio (primeros 400m).

  • Negativo: tiende a ir a más (negative split).

  • Positivo: tiende a perder ritmo.

  • kick: promedio últimos 300m (1300–1500). Negativo suele indicar kick (más - rápido al final).

  • fast_start: promedio primeros 200m (100–200). Negativo suele indicar salida agresiva.

  • variability: desviación estándar del perfil (irregularidad).

early_idx  <- split_cols %in% c("100m","200m","300m","400m")
late_idx   <- split_cols %in% c("1200m","1300m","1400m","1500m")
kick_idx   <- split_cols %in% c("1300m","1400m","1500m")
start_idx  <- split_cols %in% c("100m","200m")

strategy_metrics <- data.frame(
  Competitor = rownames(Xshape),
  cluster = factor(clusters),
  slope = rowMeans(Xshape[, late_idx, drop = FALSE]) - rowMeans(Xshape[, early_idx, drop = FALSE]),
  kick = rowMeans(Xshape[, kick_idx, drop = FALSE]),
  fast_start = rowMeans(Xshape[, start_idx, drop = FALSE]),
  variability = apply(Xshape, 1, sd)
)

2.9.2 Reglas de decisión (heurísticas)

Estas reglas usan umbrales en z-score. Son heurísticas: funcionan razonablemente, pero conviene ajustarlas si cambian el tipo de carrera, el nivel, o si quieres categorías más/menos estrictas.

Lógica general:

  • Si el final es relativamente más rápido (kick muy negativo) y además el perfil va a más (slope < 0), etiquetamos como kick final fuerte.

  • Si sale muy rápido (fast_start muy negativo) pero luego se desacelera (slope > 0), etiquetamos como salida rápida y se apaga.

  • Si el perfil es muy estable (pendiente pequeña y baja variabilidad), lo marcamos como ritmo constante.

  • Si no encaja claramente, lo dejamos como estrategia mixta.

label_strategy <- function(slope, kick, fast_start, variability) {

  if (kick < -0.5 && slope < 0) {
    return("Kick final fuerte")
  }

  if (fast_start < -0.5 && slope > 0.3) {
    return("Salida rápida, se apaga")
  }

  if (abs(slope) < 0.2 && variability < 0.6) {
    return("Ritmo muy constante")
  }

  if (slope > 0.5) {
    return("Desaceleración progresiva")
  }

  if (slope < -0.5) {
    return("Negativa progresiva")
  }

  return("Estrategia mixta")
}

strategy_metrics$label <- mapply(
  label_strategy,
  strategy_metrics$slope,
  strategy_metrics$kick,
  strategy_metrics$fast_start,
  strategy_metrics$variability
)
